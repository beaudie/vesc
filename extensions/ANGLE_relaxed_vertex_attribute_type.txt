Overview

    This extension defines behavior when the type and size parameters of
    VertexAttribPointer do not match the corresponding vertex attribute's
    type in the shader.

New Procedures and Functions

    None

New Tokens

    None

Additions to the OpenGL ES Specification

    Add after paragraph 1 of section 2.9 "Vertex Arrays":

    If VertexAttribPointer is called with type and size parameters that do not match the
    corresponding vertex attribute's type and size, the first attribute's values will be
    used for all subsequent attributes.

    If VertexAttribPointer is called with type that does not match the corresponding
    vertex attribute's type, but the VertexAttribPointer size parameters matches, the
    OpenGL should treat as if the type matched.
