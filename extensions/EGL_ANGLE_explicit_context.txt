Name

    ANGLE_explicit_context

Name Strings

    EGL_ANGLE_explicit_context

Contributors

    Brandon Jones, Intel

Contacts

    Brandon Jones, Intel (brandon1.jones 'at' intel 'dot' com)

Status

    Unimplemented Draft

Version

    Version 1, 2018-4-27

Number

    EGL Extension XXX

Extension Type

    EGL Client Extension

Dependencies

    OpenGL ES 3.0 is required.

    GL_CHROMIUM_copy_texture is required.

    GL_ANGLE_robust_client_memory is required.

Overview

    This extension adds new entry points that allow the client to pass the current EGL context as a 
    function parameter. This allows users to skip lookup of the thread's current context upon entry.

New Types

    typedef void* GLeglContextANGLE	//A handle used to pass an eglContext into gl

Additions to the EGL Specification

    When using GL ES as a client library, the glxContextANGLE entry points allow the user to 
    pass the current EGL context as a function parameter to provide better performance than
    looking the current EGL context at every GL entry point.

    Calls to "xContextANGLE" provide no additional validation for the passed context parameter.

    Calls to "xContextANGLE" will generate errors under the same conditions as "x". Any 
    instances of undefined behaviour in "x" will also be undefined in "xContextANGLE".

Issues

    1) What happens when a passed in context is not current?

    This behavior is undefined. It is the client's responsibility to ensure the passed context has 
    been set with eglMakeCurrent.
